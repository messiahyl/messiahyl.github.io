AgentMesh 使用文档

基于 Git 的 AI Agent 跨机器协同框架，Rust 实现。

快速开始

1. 编译安装

git clone https://gitee.com/messiahyl/agent-mesh-rs.git
cd agent-mesh-rs
cargo build --release --features embed-dashboard
# 二进制位置: ./target/release/agent-mesh

2. 初始化数据仓库

mkdir /tmp/agent-mesh-test && cd /tmp/agent-mesh-test
git init
mkdir -p agents channels tasks machines

3. 创建 Agent

agent-mesh --repo /tmp/agent-mesh-test agent create ceo "CEO Agent" team-lead --creator human
agent-mesh --repo /tmp/agent-mesh-test agent create dev "Dev Agent" developer --creator human

4. 启动 Dashboard

# 开发模式
AGENT_MESH_AUTH_MODE=development agent-mesh --repo /tmp/agent-mesh-test dashboard --port 3211 --listen 0.0.0.0

# 生产模式
AGENT_MESH_API_KEY=your-secret-key agent-mesh --repo /path/to/repo dashboard --port 3211
打开浏览器访问 http://localhost:3211 即可看到 Dashboard。

5. 启动同步守护进程

agent-mesh --repo /tmp/agent-mesh-test start --machine machine-a --agents ceo,dev

mesh.yaml 配置

mesh:
  name: AgentMesh
  version: "0.1.0"

sync:
  interval: 15s              # 同步间隔
  strategy: pull --rebase && push
  batch_commits: true        # 批量提交
  batch_window: 30s
  commit_prefix: "[mesh]"
  priority_paths:            # 优先同步路径
    - path: channels/
      priority: 1

conflict:
  auto_resolve: true         # 自动解决冲突

channels:
  default_visibility: team
  message_history_limit: 200

REST API

基础

方法路径说明
GET/api/health健康检查
GET/api/status全局状态概览

Agent

方法路径说明
GET/api/agents列出所有 Agent
GET/api/agents/{id}获取 Agent 详情（含 SOUL.md）
GET/api/agents/{id}/context获取完整上下文 + 系统提示词

频道

方法路径说明
GET/api/channels列出所有频道
GET/api/channels/{name}/messages获取频道消息（?limit=50）
POST/api/channels/{name}/send发送消息

任务

方法路径说明
GET/api/tasks列出所有任务
POST/api/tasks创建任务
PUT/api/tasks/{id}更新任务状态

Git

方法路径说明
GET/api/git/status仓库状态
GET/api/git/log提交历史（?count=50）
GET/api/git/tree文件树浏览
GET/api/git/diff/{commit}查看提交 diff
GET/api/git/file/{commit}?path=x查看文件内容
POST/api/git/pull手动拉取
POST/api/git/push手动推送

连接 /api/events 接收实时事件推送：

事件类型说明

task任务变更
agentAgent 状态变更
sync同步完成

AI Agent 使用指南

完整协作工作流

  1. CEO 创建任务 — POST /api/tasks {title, assign}
  2. 系统广播事件 — SSE 推送 new_task 事件
  3. Dev 接收通知 — 更新状态为 in_progress
  4. Dev 执行工作 — 定期在频道更新进度
  5. Dev 提交审核 — 状态 → review，@mention CEO
  6. CEO 审核关闭 — 状态 → done

关键 API 调用

# 获取上下文（最重要）
curl http://localhost:3211/api/agents/ceo/context?machine=machine-a

# 发送消息
curl -X POST http://localhost:3211/api/channels/general/send \
  -H "Content-Type: application/json" \
  -d '{"from":"ceo","content":"@dev 任务已分配","mentions":["dev"]}'

# 更新任务
curl -X PUT http://localhost:3211/api/tasks/TASK-001 \
  -H "Content-Type: application/json" \
  -d '{"status":"in_progress"}'

常见问题

工作区变更显示了整个 home 目录？

需要指定 --repo 参数指向正确的 Git 数据仓库：

agent-mesh --repo /path/to/data-repo dashboard --port 3211

如何在两台机器间协作？

  1. 两台机器 clone 同一个 Git 仓库
  2. 分别启动 agent-mesh start --machine <各自机器名> --agents <各自的Agent>
  3. SyncDaemon 会自动 Pull-Push 保持同步

冲突如何处理？

AgentMesh 自动处理三种冲突：

  • state.yaml：合并 instances 字段（取并集）
  • 频道消息：保留 ours（theirs 会在下次同步重新出现）
  • 身份文件：取 theirs（远程优先）
